# ADR-013

---

## Título

Lenguaje y framework principales para el desarrollo del backend cloud.

## Fecha

11/04/2026

## Estado

Tomada

---

## Contexto

El backend debe ser capaz de establecer una comunicación con los clientes a través del protocolo WebSocket, así como ofrecer endpoints REST. A su vez es necesario elegir un lenguaje que permita fácilmente interactuar con una base de datos PostgreSQL, con S3, Redis, SQS e IoT Core en AWS, y que ofrezca servicios de autenticación conocidos para una sencilla implementación del componente de autenticación personalizado.

## Decisión tomada

Construir el backend usando **Python**, particularmente con el framework **FastAPI**.

---

## Factores decisivos

1. Python es ampliamente usado, teniendo varios frameworks populares, miles de librerías y documentación extensa sobre su uso. Además, todo el equipo ya es extremadamente familiar con el lenguaje.   
2. La arquitectura demanda tanto una API REST como una API WebSocket; y FastAPI soporta ambas de manera nativa sin necesidad de agregar librerías externas ni configuración extra.  
3. Es un framework pensado para la comunicación asíncrona, la cual nuestro sistema utiliza extensivamente.  
4. Opciones como la validación automática de tipos de datos mediante Pydantic o la generación de documentación OpenAPI son puntos a favor que pueden hacer más ameno el desarrollo.

## Alternativas

- **Flask***:* Es simple y muy conocido, pero no fue diseñado para ser asíncrono. El soporte de WebSockets requiere dependencias adicionales como `flask-socketio`, lo que introduce complejidad innecesaria y una integración forzada de la comunicación en tiempo real.  
- **Django \+ Django Channels***:* Ofrece WebSockets a través de Channels y es un framework muy completo, pero está diseñado alrededor de su propio gestor de datos relacionales y sistema de autenticación, ninguno de los cuales se usaría aquí. Es excesivo para nuestra escala.

## Pros y Contras de Python \+ FastAPI

**Pros:**

- Soporte nativo de REST y WebSockets.  
- Alto rendimiento en comunicaciones asíncronas.  
- Validación y serialización automática de datos con Pydantic.  
- Sistema de dependencias `Depends` muy útil, especialmente para la autenticación.

**Contras:**

- Al ser más reciente, hay menos documentación y estándares que los disponibles para Flask o Django.
