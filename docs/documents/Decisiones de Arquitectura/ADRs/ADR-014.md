# ADR-014

---

## Título

Selección de un cliente PostgreSQL para Python, y una forma sencilla de gestionarlo.

## Fecha

11/04/2026

## Estado

Tomada

---

## Contexto

Las interacciones con la base de datos relacional deben hacerse desde el componente `Persistencia`, que forma parte del backend y por lo tanto será escrito en Python. Es entonces necesario elegir una herramienta y driver capaces de mapear datos relacionales a Python, que a su vez deben poder funcionar de manera asíncrona para no bloquear el backend mientras se espera una respuesta de la base de datos..

## Decisión tomada

Basar las interacciones con la base de datos relacional en SQLAlchemy Core (No ORM) con psycopg3 de driver.

---

## Factores decisivos

1. SQLAlchemy Core permite escribir consultas SQL de manera explícita y directa, lo que es apropiado para un sistema donde las queries pueden ser específicas y son conocidas de antemano. No se necesita la abstracción completa de un ORM.  
2. Psycopg3 sigue el estándar DB-API 2.0 de Python, que es la interfaz que se esperaría encontrar en un cliente de base de datos de Python. Esto significa que el código se lee y se escribe de forma natural, y familiarizarse con este estándar es valioso y útil para el equipo.  
3. La integración con SQLAlchemy es directa, simplemente se especifica psycopg como driver en `connection string` y funciona.

## Alternativas

- **SQLAlchemy ORM**: Añade una abstracción al mapear entidades a clases de Python. Para este sistema, donde las consultas son relativamente directas y el equipo tiene control total del esquema, dicha abstracción agrega complejidad sin beneficios claros.  
- **psycopg3 solo**: Da control total sobre las queries. Sin embargo, obliga a gestionar manualmente la conexión y otros detalles que SQLAlchemy resuelve y simplifica.  
- **SQLAlchemy Core con asyncpg**: Asyncpg es el driver asíncrono más rápido para PostgreSQL en Python; sin embargo, tiene una API no estándar que no sigue DB-API 2.0, y en su lugar utiliza su propia API ligeramente más compleja, e inconveniente. Para la escala y el contexto de este proyecto, la diferencia de rendimiento que nos daría es despreciable.

## Pros y Contras de SQLAlchemy Core con psycopg3

**Pros:**

- SQLAlchemy simplifica y abstrae enormemente todo el proceso y mantenimiento de la conexión a la base de datos, así como las interacciones.  
- SQLAlchemy Core permite formular las queries con comandos de SQL, facilitando la generación de las consultas y mejorando la experiencia de desarrollo.  
- Psycopg3 ofrece cliente sync y async en el mismo paquete, con la misma interfaz.  
- API familiar y estándar (DB-API 2.0).

**Contras:**

- Levemente más lento que asyncpg, diferencia que en la práctica no tiene impacto para este sistema.
